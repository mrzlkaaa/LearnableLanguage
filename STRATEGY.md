### 1.2 Критические проблемы кода (из ANALYSIS_telegram_ux.md)

🔴 **P0 — блокируют:**
- Дублирование слов при ошибке (O(N²) + утечка памяти)
- Race condition в `card_counter` (FSM data неатомарно)
- Нет позиции в UI ("Карточка 3 из 10")

🟡 **P1 — сильно влияют на UX:**
- Кнопка "Предыдущая" всегда активна
- "Назад" не сохраняет позицию в карточках
- "Карточки закончились" без статистики

🟢 **P2 — улучшения:**
- `reading_quiz` state не используется
- History tracking хрупкий
- Нет лоадеров

### 1.3 Конкуренты

| Конкурент | Сильные стороны | Слабые стороны (наша ниша) |
|-----------|-----------------|---------------------------|
| **Duolingo** | Геймификация, красивый UI, привычка | "Заканчиваешь урок — забываешь" (SRS нет), нет контекста |
| **Anki** | SRS, открытый | Ужасный UX, нет AI, не для Telegram |
| **Memrise** | Контекст, видео | Платная, нет Telegram |
| **BotMother / Other Telegram bots** | Простые карточки | Нет AI, нет продуманного SRS |

### 1.4 Наше УТП (уникальное торговое предложение)

> **Контекстное Интервальное Повторение (CSRS)** — AI-генерируемые слова со сменой механики проверки на каждом уровне SRS (Use Case → Multiple Choice → Reverse Translation → Audio → Voice Production).

Документ `Общее_по_vocabulary.docx` уже описывает это УТП. Наша задача — **реализовать** это, а не придумать.

#### A5: Полноценный онбординг — что нужно реализовать

**Текущий онбординг (onboarding.py) умеет:**
- ✅ Принимает цель (через text, а не кнопки)

## 5. ЧТО МОЖНО УЛУЧШИТЬ В ТЕКУЩИХ РЕШЕНИЯХ

### 5.1 WordsSupplyService (words_supply.py)
**Проблема:** `ai_service = self.ai_service(gemini_client)` — вызов как конструктора! Это TypeError, ai_service уже объект.

**Решение:**
```python
# Передавать готовый ai_service в __init__ или исправить логику
gemini_client = GeminiClient(api_key=config.GEMINI_API_KEY.get_secret_value())
```

### 5.2 WordsLearningService (words_learning.py)
**Проблема:** `get_new_global_words` возвращает `list[WordCardDTO] | None` — None если слов нет, но `len(words) > 0` проверит длину у None → TypeError.

**Решение:**
```python
async def get_new_global_words(self, user_id: int, num_words: int = 100) -> list[WordCardDTO]:
    ...
    if len(words) == 0:
        return []  # Не None!
```

### 5.3 NotificationManager
**Проблема:** Создаёт новые сессии и сервисы в цикле — дорого.

**Решение:** Вынести создание сессий наружу, использовать connection pool.

### 5.4 Общая архитектура
**Проблема:** Handler напрямую работает с repo, создаёт service вручную. Нет DI через middleware.

**Решение:** Внедрить dependency injection (как делается с user_repo через middleware).
